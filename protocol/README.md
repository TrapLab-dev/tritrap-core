# Protocol Layer

The protocol directory defines the core execution contracts of the TriTrap architecture.
These protocol files describe invention and interface structure.
They should not be treated as sole present-tense proof of the current canonical live runtime.

These specifications describe the artifacts and procedures used to authorize, execute, and verify inference workloads.

Core components:

• Permit — execution authorization artifact  
• Receipt — execution proof artifact  
• Gate Protocol — request authorization and routing  
• Lane Execution — execution environment responsibilities  
• Arbitration — verification and result resolution
